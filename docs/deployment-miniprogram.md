# 微信小程序版 — 本地部署与上架完整教程

本文覆盖两件事:**本地开发运行** `miniprogram/`,以及**个人开发者最低成本上架**。
NAS 版(前端 + 后端)的部署见 [`deployment.md`](deployment.md);两者独立,互不影响。

---

## 一、准备

1. 安装[微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)(稳定版)。
2. (上架才需要)在 [mp.weixin.qq.com](https://mp.weixin.qq.com) 注册小程序账号,拿到 **AppID**。
   - 仅本地体验可用工具内置的「测试号」,无需注册。

---

## 二、本地开发运行

1. 打开微信开发者工具 → **导入项目** → 目录选择仓库里的 `miniprogram/`。
2. AppID 填你自己的,或点「测试号」。`project.config.json` 里默认是 `touristappid`。
3. 首次进入会落在「助手」页。点底部 Tab 可切到 物品 / 位置 / 平面图 / 备份。
4. **关闭域名校验**(开发期):右上角「详情 → 本地设置 → 勾选『不校验合法域名、TLS 版本以及 HTTPS 证书』」。
   这样才能在开发阶段连任意 LLM / WebDAV。

### 配置 LLM(用于「助手」语义指令)

进「助手」页右下角 **⚙** 进入设置:
1. 点一个预设(硅基流动 / DeepSeek / 智谱 GLM / OpenAI)或手填 Base URL。
2. 填 API Key、Model。
3. 点 **测试 LLM**,返回一段文字即通。
4. 保存。

> ⚠️ 小程序是纯客户端,API Key 存在用户设备本地。**定位为个人自用**;若要公开分发,需另搭代理(那会引入后端,与"纯本地"相悖)。

### 启用语音(可选,需「同声传译」插件)

1. 上架主体下:mp 后台 →「设置 → 第三方设置 → 插件管理 → 添加插件」,搜索 **微信同声传译** 添加。
2. `app.json` 已声明该插件(`plugins.WechatSI`)。添加后,「助手」页的 🎙 按钮即可长按说话。
3. 未添加插件时,语音按钮自动隐藏,**文本输入始终可用**。

### 配置 WebDAV 备份

进「备份」页:
1. 填服务器地址(如坚果云 `https://dav.jianguoyun.com/dav/`)、账号、应用密码、远程子目录。
2. **先在网盘里手动创建该子目录**(小程序受微信限制不能自动建目录)。
3. 加密口令在「设置」页填(留空=不加密;填了与 NAS 版 AES-256 互通)。
4. 「立即备份」→ 网盘出现 `backup-*.zip`(加密则 `*.enc.zip`);列表/恢复/删除均可用。

### 真机预览

点工具栏「预览」生成二维码,微信扫码即可在手机上体验(注意真机会校验合法域名,见下)。

## 三、上架(个人开发者,约 ¥30/年)

### 费用

| 项 | 费用 |
|---|---|
| 注册账号 | ¥0 |
| 个人主体认证 | **¥30/年**(一年一审) |
| 服务器/域名 | ¥0(纯本地;LLM/WebDAV 用第三方已备案的可信 HTTPS 域名) |

### 步骤

1. **个人认证**:mp 后台「设置 → 基本设置」按引导完成个人身份认证(身份证 + 扫脸,¥30)。不认证则小程序无法被搜索、不能分享。
2. **填基本信息**:名称、头像、简介、服务类目(选「工具」)。简介如实写「个人/家庭物品收纳管理工具」。
3. **配置合法域名**:mp 后台「开发 → 开发管理 → 开发设置 → 服务器域名」,在 **request 合法域名** 里加入:
   - 你的 **LLM API 域名**(如 `https://api.siliconflow.cn`)
   - 你的 **WebDAV 域名**(如 `https://dav.jianguoyun.com`)
   - 要求 HTTPS + 可信证书(**自签证书不可用**);每月有修改次数限制。
4. **填 AppID**:把工具里项目的 AppID 换成你自己的(「详情 → 项目设置」或重新导入)。
5. **上传**:开发者工具点「上传」→ 填版本号(如 `1.0.0`)与备注 → 进入 mp 后台「管理 → 版本管理」。
6. **提交审核**:在「版本管理 → 开发版本」点「提交审核」。纯本地工具、无内容分发,审核风险低;审核约 **7–10 个工作日**。
7. **发布**:审核通过后点「发布」,用户即可在微信里搜到并使用。

### 个人号限制(对本项目影响很小)

- 无微信支付、无直播、部分开放能力受限 —— 本项目都不用。
- 一年一审:到期前完成年审,否则搜索/分享失效。

## 四、与 NAS 版数据互通

两端备份包同格式,可一份数据两端用:

| 操作 | 结果 |
|---|---|
| 小程序备份 → NAS 版「备份」页「从文件恢复」上传,或放进同一 WebDAV 目录后恢复 | ✅ NAS 后端自动从 `data/*.json` 重建(已验证) |
| NAS 版备份 → 小程序「备份」页恢复 | ✅ 小程序能解 NAS 的 DEFLATE 压缩包(已实现 inflate) |
| 加密备份两端互通 | ✅ AES-256-GCM 同口令,字节级互通(已验证) |

> 小程序与 NAS 版连同一个 WebDAV 目录即可共享备份历史(小程序靠目录里的 `index.json` 列表;
> 若只在 NAS 版备份,小程序列表为空属正常,可用「从文件恢复」或先在小程序备份一次以生成 index.json)。

## 五、已知限制(MVP)

- 本地存储:小数据 `wx.storage`(10MB),大表(物品/流水/审计)走 `FileSystemManager`(200MB)。
- WebDAV:`wx.request` 不支持 `PROPFIND`/`MKCOL` → 列表用远程 `index.json`,目录需手动预建。
- 3D:当前为 **2D 平面图**(Canvas 2D);完整 Three.js 3D 属后续阶段。
- 语音:依赖「同声传译」插件;文本指令始终可用作为兜底。
